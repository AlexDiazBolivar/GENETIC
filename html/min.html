genetic.min

Main interface for minimisation.

All the optimisation method in Genetic are called through the unified interface genetic.min. A similar interface exists for maximisation: genetic.max and is used in the exact same way.

Syntax

[xopt, fopt, info] = genetic.min(f, x, method)
[xopt, fopt, info] = genetic.min(f, x, method, options)
[xopt, fopt, info] = genetic.min(f, x, method, constraints, options)

Arguments

Inputs

f The objective function. More specifically f can either be
  • a handle function,
  • a cell containing a handle function and the number of objectives, i.e. f = {f_handle, fDim} (mandatory for multi-objective optimisation),
  • a string with the name of a benchmark available in genetic.
x Dimension of the problem or an initial point/population.
method Name of the method to be used. See the available methods.
options A structure containing the parameters for the optimisation process. See the options section for further details.
constraints A structure containing a description of the constraints of the optimisation problem.

Outputs

xopt Best solution(s) found.
fopt Associated objective of Pareto front.
info Structure containing informations about the optimisation process.

Available methods

The mono and multi-objective methods that are included in the Genetic toolbox are listed below. Note that those algorithms may belong to different classes and may therefore have very different properties. For instance an algorithm may be stochastic or deterministic, require the gradient of the objective function or not, etc.

Mono-objective

Key Name of the algorithm Stochastic Derivative-free
annealing Simulated annealing ✔ ✔
pso Particle Swarm Optimisation algorithm ✔ ✔
cmaes Covariance Matrix Adaptation Evolution Strategy ✔ ✔
simplex Nelder-Mead Simplex algorithm ✔
gss Generating Set Search Method ✔
linesearch Local descent algorithm based on line-search

Multi-objective

Key Name of the algorithm Stochastic Derivative-free
nsga2 Non-dominated Sorting Genetic Algortihm II ✔ ✔
nsga3 Non-dominated Sorting Genetic Algorithm III ✔ ✔
mopso Multi-Objective Particle Swarm Optimization ✔ ✔

Options

The optional parameters are divided into three categories:

  • general parameters that may affect all the algorithms,
  • class-specific parameters that may affect either mono or multi-objective methods,
  • method-specific parameters that will affect only some method.

General and class-specific parameters are listed below with their default values while method-specific parameters are described in the help page of the corresponding method.

General

Key Description Default value
maxFunEval Maximum number of function evaluation 5000
verbosity Verbosity level of the algorithm 0
popSize Dimension of the population (for population based algorithms) round(10+2*sqrt(n))
gradObj Availability of the gradient as second output of the objective function false

Specific to mono-objective methods

Key Description Default value
targetY Target value of the objective function below which it is not useful to go -inf
tolY Minimum decrease tolerated between two consecutive best objectives 1e-8
ntolY Number of times the minimum objective decrease must be met 1
tolX Minimum distance tolerated between two consecutive best solutions 1e-8
ntolX Number of times the minimum distance must be met 1

Constraints

Similarly to options, constraints are added through a structure containing specific fields depending on the constraints:

  • bounds are characterized by the fields xMin and xMax such that: xmin ≤ x ≤ xmax,
  • linear inequality are characterized by the fields A and b such that: Ax ≤ b,
  • linear equality are characterized by Aeq and beq such that: Aeqx=beq,
  • non-linear inequality constraints are given as an anonymous function c such that: c(x) ≤ 0,
  • non-linear equality constraints are given as an anonymous function ceq such that: ceq(x) = 0.

Examples

This section contains two trivial examples aimed at illustrating the calling sequence, for more real-world use cases, refer to the dedicated page.

Mono-objective

Minimisation of the square of the 2-norm in dimension 10 with the cmaes method and with a maximum of 3000 allowed function evaluations. 

xDim                = 10;
f                   = @(x) norm(x)^2;
method              = 'cmaes';
options             = struct('verbosity', 3, 'maxFunEval', 3000);
[xopt, fopt, info]  = genetic.min(f, xDim, method, options);

Multi-objective

Minimisation of a function with 2 objectives with the nsga2 method with 10000 maximum functions evaluations. Note that the dimension of the objective function must be specified in that case. 

xDim                = 10;
A                   = rand(xDim,xDim);
b                   = rand(xDim,1);
fun                 = @(x) [norm(x)^2;norm(A*x - b)^2];
fDim                = 2;
f                   = {fun, fDim};
method              = 'nsga2';
options             = struct('verbosity', 3, 'maxFunEval', 10000);
[xopt, fopt, info]  = genetic.min(f, xDim, method, options);
